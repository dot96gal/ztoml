# Zig 移行計画：v0.15.2 → v0.16.0

## 概要

`mise.toml` のバージョンは既に `zig = "0.16.0"` に更新済み。
コードベースの移行作業を行う。

## スキャン結果

### 問題なし（移行済み）

- `src/` 配下（`parser.zig`, `types.zig`, `deserialize.zig`, `mod.zig`, `root.zig`）
  - `std.ArrayListUnmanaged` を使用済み → 0.16 互換
  - ファイル IO なし → `std.Io` 影響なし
- `e2e/main.zig` — ファイル IO なし、テスト専用
- `examples/basic.zig` — `std.debug.print` のみ

### 要修正（bench/main.zig）

| 行 | 問題 | 対応 |
|----|------|------|
| 6  | `pub fn main() !void` — io 取得不可 | `pub fn main(env: std.process.Init) !void` に変更 |
| 12–13 | `std.process.argsAlloc` / `argsFree` | `env.minimal.args.toSlice(env.arena.allocator())` に変更 |
| 16 | `std.fs.cwd()` → 廃止 | `std.Io.Dir.cwd()` に変更、ファイル読み込みに `io` を使う |
| 25 | `std.time.Timer` → 廃止 | `std.Io.Timestamp` に変更 |
| 30 | `timer.read()` | `std.Io.Timestamp` の API に合わせて変更 |

## 修正方針

```zig
pub fn main(env: std.process.Init) !void {
    const allocator = env.gpa;
    const io = env.io;
    const raw_args = try env.minimal.args.toSlice(env.arena.allocator());

    const path = if (raw_args.len > 1) raw_args[1] else "testdata/large.toml";

    const cwd = std.Io.Dir.cwd();
    const input = try cwd.readFileAlloc(io, allocator, path, 10 * 1024 * 1024);
    defer allocator.free(input);

    // Warm up
    for (0..10) |_| {
        var result = try toml.parseFromSlice(allocator, input, .{});
        result.deinit();
    }

    const start = io.timestamp();
    for (0..RUNS) |_| {
        var result = try toml.parseFromSlice(allocator, input, .{});
        result.deinit();
    }
    const elapsed_ns = io.timestamp().since(start);
    // ...
}
```

> **注意**：`std.Io.Dir.cwd().readFileAlloc` の正確なシグネチャは 0.16 ドキュメントで要確認。
> `std.Io.Timestamp` の差分取得 API（`since` / `-` 演算子等）も確認が必要。

## 振り返り：計画との差異

### 1. スキャン結果の見落とし

計画では以下を「問題なし」と判断したが、実際は修正が必要だった。

| ファイル | 見落とし内容 |
|---------|-------------|
| `src/toml/parser.zig` | `ArrayListUnmanaged` の `.{}` 初期化が 0.16 で非互換（→ `.empty` に変更）。8箇所（変数宣言6・インライン初期化2）。 |
| `examples/basic.zig` | `std.heap.GeneralPurposeAllocator` が 0.16 で廃止（→ `env.gpa` に変更）。`std.debug.print` のみと誤判断した。 |

### 2. `readFileAlloc` のシグネチャ相違

計画のコード例（注意書きあり）と実際の 0.16 シグネチャが異なった。

```zig
// 計画（誤）
cwd.readFileAlloc(io, allocator, path, 10 * 1024 * 1024)

// 実際（正）
cwd.readFileAlloc(io, path, allocator, std.Io.Limit.limited(10 * 1024 * 1024))
```

- `path` と `allocator` の順序が逆
- 第4引数は `usize` ではなく `std.Io.Limit` 型（`Io.Limit.limited(n)` で生成）

### 3. タイムスタンプ API の相違

計画では `io.timestamp()` / `.since()` と記載したが、`Io` に `timestamp` メソッドは存在しない。

```zig
// 計画（誤・存在しない API）
const start = io.timestamp();
const elapsed_ns = io.timestamp().since(start);

// 実際（正）
const start = std.Io.Clock.Timestamp.now(io, .awake);
const elapsed_ns: u64 = @intCast(start.durationTo(std.Io.Clock.Timestamp.now(io, .awake)).raw.toNanoseconds());
```

- クロック種別を明示（`.awake` = モノトニック、システムサスペンド時間除外）
- 差分は `durationTo` で `Clock.Duration` を取得し、`.raw.toNanoseconds()` で `i96` → `u64` にキャスト
