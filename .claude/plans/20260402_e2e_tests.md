# E2Eテスト実装計画

作成日: 2026-04-02

## 目的

TOML v1.1.0 仕様の各セクションに対応するE2Eテストを実装し、仕様変更や内部実装変更時のリグレッションテストとして活用できるようにする。

## 方針

- `bench/` ディレクトリのベンチマーク構成に倣い、`e2e/` ディレクトリにE2Eテストを配置する
- `e2e/main.zig` にZigのテストブロック（`test "..." { ... }`）を実装する
- `build.zig` に `e2e` ステップを追加し、`zig build e2e` で実行できるようにする
- TOMLの仕様ごとに1テストケースを作成する（既存の単体テストとは分離）
- テストはライブラリの公開API（`ztoml.parseFromSlice`）のみを使用する

## ディレクトリ構成

```
/
├── bench/
│   └── main.zig        # 既存ベンチマーク（参考）
├── e2e/
│   └── main.zig        # E2Eテスト（新規作成）
├── src/
│   └── ...             # 既存ソース（単体テスト含む）
└── build.zig           # e2e ステップを追加
```

## build.zig への変更

benchステップと同様に、`e2e/main.zig` をテスト実行ファイルとして追加する。

```zig
// E2Eテスト
const e2e_tests = b.addTest(.{
    .root_module = b.createModule(.{
        .root_source_file = b.path("e2e/main.zig"),
        .target = target,
        .optimize = optimize,
        .imports = &.{
            .{ .name = "ztoml", .module = mod },
        },
    }),
});
const run_e2e_tests = b.addRunArtifact(e2e_tests);
const e2e_step = b.step("e2e", "Run E2E tests");
e2e_step.dependOn(&run_e2e_tests.step);
```

## e2e/main.zig のテストケース一覧

TOML v1.1.0 仕様の各セクションに対応するテストを実装する。

### 1. Comment（コメント）

仕様: `#` 以降は行末までコメント。文字列内の `#` はコメントではない。

```toml
# This is a full-line comment
key = "value"  # This is a comment at the end of a line
another = "# This is not a comment"
```

期待: `key = "value"`, `another = "# This is not a comment"`

### 2. Key/Value Pair（キーと値のペア）

仕様: キーと値は `=` で区切られる。同一行に複数のキーと値のペアは無効。

```toml
key = "value"
```

期待: `key = "value"`

### 3. Keys - Bare Keys（ベアキー）

仕様: ASCII文字・数字・アンダースコア・ハイフンのみ使用可能。数字のみも有効。

```toml
key = "value"
bare_key = "value"
bare-key = "value"
1234 = "value"
```

期待: 各キーが文字列値として正しく解析される

### 4. Keys - Quoted Keys（クォートキー）

仕様: ベーシック文字列またはリテラル文字列のルールに従う。

```toml
"127.0.0.1" = "value"
"character encoding" = "value"
'key2' = "value"
'quoted "value"' = "value"
```

期待: 各キーが正しく解析される

### 5. Keys - Dotted Keys（ドットキー）

仕様: ドットで区切られた一連のベアキーまたはクォートキー。テーブルを定義する。

```toml
name = "Orange"
physical.color = "orange"
physical.shape = "round"
site."google.com" = true
```

期待JSONに相当する構造:
```json
{
  "name": "Orange",
  "physical": { "color": "orange", "shape": "round" },
  "site": { "google.com": true }
}
```

### 6. String - Basic String（ベーシック文字列）

仕様: ダブルクォートで囲まれた文字列。エスケープシーケンス対応。

```toml
str = "I'm a string. \"You can quote me\". Name\tJos\xE9\nLocation\tSF."
```

期待: エスケープが正しく展開された文字列

### 7. String - Multi-line Basic String（複数行ベーシック文字列）

仕様: `"""` で囲まれた文字列。開始直後の改行はトリム。行末バックスラッシュで空白をトリム。

```toml
str1 = """
Roses are red
Violets are blue"""
str2 = """
The quick brown \

  fox jumps over \
    the lazy dog."""
```

期待: `str1 = "Roses are red\nViolets are blue"`, `str2 = "The quick brown fox jumps over the lazy dog."`

### 8. String - Literal String（リテラル文字列）

仕様: シングルクォートで囲まれた文字列。エスケープなし。

```toml
winpath  = 'C:\Users\nodejs\templates'
winpath2 = '\\ServerX\admin$\system32\'
quoted   = 'Tom "Dubs" Preston-Werner'
regex    = '<\i\c*\s*>'
```

期待: バックスラッシュがそのまま保持される

### 9. String - Multi-line Literal String（複数行リテラル文字列）

仕様: `'''` で囲まれた文字列。開始直後の改行はトリム。エスケープなし。

```toml
regex2 = '''I [dw]on't need \d{2} apples'''
lines  = '''
The first newline is
trimmed in literal strings.
   All other whitespace
   is preserved.
'''
```

期待: エスケープが行われず、内容がそのまま保持される

### 10. Integer - Decimal（10進整数）

仕様: 正数はプラス符号省略可能。負数はマイナス符号。アンダースコアで区切り可能。

```toml
int1 = +99
int2 = 42
int3 = 0
int4 = -17
int5 = 1_000
int6 = 5_349_221
```

期待: 各整数値が正しく解析される

### 11. Integer - Non-decimal（非10進整数）

仕様: 16進数（`0x`）、8進数（`0o`）、2進数（`0b`）プレフィックス対応。

```toml
hex1 = 0xDEADBEEF
hex2 = 0xdeadbeef
oct1 = 0o01234567
oct2 = 0o755
bin1 = 0b11010110
```

期待: 各数値が正しい10進数値に変換される

### 12. Float - Fractional and Exponent（浮動小数点数）

仕様: 小数部またはべき乗部を持つ浮動小数点数。アンダースコア区切り可能。

```toml
flt1 = +1.0
flt2 = 3.1415
flt3 = -0.01
flt4 = 5e+22
flt5 = 1e06
flt6 = -2E-2
flt7 = 6.626e-34
flt8 = 224_617.445_991_228
```

期待: 各浮動小数点数値が正しく解析される

### 13. Float - Special Values（特殊浮動小数点数）

仕様: `inf`, `+inf`, `-inf`, `nan`, `+nan`, `-nan` が有効な値。

```toml
sf1 = inf
sf2 = +inf
sf3 = -inf
sf4 = nan
sf5 = +nan
sf6 = -nan
```

期待: 各特殊値が正しく解析される（inf は無限大、nan は非数）

### 14. Boolean（ブール値）

仕様: `true` と `false` のみ。常に小文字。

```toml
bool1 = true
bool2 = false
```

期待: `bool1 = true`, `bool2 = false`

### 15. Offset Date-Time（オフセット付き日時）

仕様: RFC 3339形式。`T`の代わりにスペース使用可能。秒省略可能。

```toml
odt1 = 1979-05-27T07:32:00Z
odt2 = 1979-05-27T00:32:00-07:00
odt3 = 1979-05-27T00:32:00.999999-07:00
odt4 = 1979-05-27 07:32:00Z
odt5 = 1979-05-27 07:32Z
```

期待: 各日時値が正しく解析される

### 16. Local Date-Time（ローカル日時）

仕様: オフセットなしのRFC 3339形式。秒省略可能。

```toml
ldt1 = 1979-05-27T07:32:00
ldt2 = 1979-05-27T07:32:00.5
ldt3 = 1979-05-27T07:32
```

期待: 各ローカル日時値が正しく解析される

### 17. Local Date（ローカル日付）

仕様: RFC 3339形式の日付部分のみ。

```toml
ld1 = 1979-05-27
```

期待: 日付が正しく解析される

### 18. Local Time（ローカル時刻）

仕様: RFC 3339形式の時刻部分のみ。秒省略可能。

```toml
lt1 = 07:32:00
lt2 = 00:32:00.999999
lt3 = 07:32
```

期待: 各時刻値が正しく解析される

### 19. Array（配列）

仕様: 角括弧で囲まれた値のリスト。異なる型の混在可能。複数行対応。末尾カンマ許容。

```toml
integers = [ 1, 2, 3 ]
colors = [ "red", "yellow", "green" ]
nested_arrays_of_ints = [ [ 1, 2 ], [3, 4, 5] ]
numbers = [ 0.1, 0.2, 0.5, 1, 2, 5 ]
integers3 = [
  1,
  2, # this is ok
]
```

期待: 各配列が正しく解析される

### 20. Table（テーブル）

仕様: `[テーブル名]` ヘッダで定義。その後、次のヘッダまたはEOFまでがキーと値のペア。

```toml
[table-1]
key1 = "some string"
key2 = 123

[table-2]
key1 = "another string"
key2 = 456
```

期待: 各テーブルが正しく解析され、階層構造が正しく構築される

### 21. Inline Table（インラインテーブル）

仕様: `{...}` で表現される。複数行対応（v1.1.0）。末尾カンマ許容（v1.1.0）。

```toml
name = { first = "Tom", last = "Preston-Werner" }
point = {x=1, y=2}
animal = { type.name = "pug" }
contact = {
    personal = {
        name = "Donald Duck",
        email = "donald@duckburg.com",
    },
    work = {
        name = "Coin cleaner",
        email = "donald@ScroogeCorp.com",
    },
}
```

期待: 各インラインテーブルが正しく解析される

### 22. Array of Tables（テーブルの配列）

仕様: `[[テーブル名]]` ヘッダで定義。同名ヘッダごとに配列に要素が追加される。

```toml
[[products]]
name = "Hammer"
sku = 738594937

[[products]]

[[products]]
name = "Nail"
sku = 284758393
color = "gray"
```

期待: `products` が3要素の配列として解析される

```bash

# mise タスク経由（mise.toml に追加）
mise run e2e
```
